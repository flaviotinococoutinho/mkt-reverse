## 2024-05-18 - Adding ARIA labels to dynamically generated form lists
**Learning:** When implementing accessibility improvements for dynamic lists of inputs (like the key-value attribute editors), adding generic aria-labels like "Remove attribute" isn't enough, because a screen reader will announce several identical "Remove attribute" buttons on the page. Interpolating the item's identifying value into the `aria-label` (e.g. `Remover atributo ${attr.key || 'novo'}`) gives the screen reader the specific context it needs to identify which list item action is being invoked.
**Action:** Always interpolate the item's identifying value into the `aria-label` for list item actions (e.g. "Remove attribute Size") to provide specific context for screen readers.
