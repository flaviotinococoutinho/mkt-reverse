# 2026-02-07 - Implementação do MVP do Marketplace Reverso (mkt-reverse)

## 🎨 Frontend (React/Vite)
- **Componentes Criados:**
    - `AttributeEditor.tsx`: Editor visual para atributos de produtos (`SpecAttribute`), eliminando a necessidade de editar JSON manualmente. Suporta tipos como TEXT, NUMBER, VOLTAGE, COLOR, etc.
    - `AuthContext.tsx`: Contexto de autenticação para gerenciar `tenantId`, `buyerOrganizationId` e informações de contato globalmente.
- **Páginas Atualizadas:**
    - `CreateEventPage.tsx`: Integrada com `AuthContext` e `AttributeEditor`. Preenchimento automático de campos do comprador e desabilitação de campos quando o usuário está autenticado.
    - `main.tsx`: Configurado para envolver a aplicação no `AuthProvider` com um `MOCK_USER` padrão para facilitar o desenvolvimento local.

## ⚙️ Backend (Java 21/Spring Boot)
- **Módulo Shared:**
    - `Money.java`: Adicionado método estático `fromCents(long cents, CurrencyCode currency)` para encapsular a lógica de conversão de centavos e evitar divisões manuais em controladores/serviços.
- **Módulo Contract Management:**
    - `ContractApplicationService.java`: Implementado o serviço de aplicação para gerenciar a criação de contratos.
    - `ContractSourcingListener.java`: Implementado listener `@EventListener` para reagir ao evento `SourcingEventStatusChangedEvent`. Agora, quando um evento de sourcing é marcado como `AWARDED`, um rascunho de contrato é gerado automaticamente no banco de dados.

## 📋 Arquitetura e MVP
- **Alinhamento:** Foram resolvidas críticas de code review ("Cron report") sobre hardcoded auth, validação manual de JSON e falta de integração entre módulos de Sourcing e Contratos.
- **Status:** O fluxo principal "Reverse Marketplace" (Criação de Pedido -> Proposta -> Aceite -> Geração de Contrato) está tecnicamente completo no backend e robusto no frontend.
